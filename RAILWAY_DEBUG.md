# Railway: чому бот не працює після редеплою

## 1. Дивись **Deploy Logs**, не Build Logs

- **Build Logs** — тільки збірка образу (Dockerfile). У тебе вона пройшла (5.98 s).
- **Deploy Logs** — що реально робить контейнер при старті. Саме там буде помилка.

Відкрий вкладку **Deploy Logs** і знайди:
- рядки `Frog bot: starting...` та `config loaded` — значить старт пройшов до завантаження конфігу;
- якщо є **RuntimeError: BOT_TOKEN is not set** — токен не передано в Variables;
- будь-який **Traceback** — покаже, де саме падає бот.

## 2. Variables (обовʼязково)

У проєкті **leinerfrog-bot** → **Variables** мають бути:

| Змінна | Обовʼязкова | Приклад |
|--------|-------------|--------|
| `BOT_TOKEN` | ✅ Так | токен від @BotFather |
| `DB_PATH` | Ні | `leinerfrog.db` (за замовчуванням) |
| `OPENWEATHER_API_KEY` | Ні | для погоди о 8:00 |
| `WEATHER_CITY` | Ні | `Kyiv` |
| `HOST_USER_ID` | Ні | Telegram user_id хазяїна |

Якщо **BOT_TOKEN** порожній або його немає, `load_config()` кидає виключення і контейнер одразу завершується.

## 3. Root Directory

Якщо репо на GitHub має бота в підтеці (наприклад `frogbot/`):

- У Railway → **Settings** → **Root Directory** вкажи цю папку (наприклад `frogbot`), або
- переконайся, що **Dockerfile** і **requirements.txt** лежать у тій директорії, яку Railway використовує як корінь збірки.

## 4. Що робити крок за кроком

1. **Variables** → переконайся, що є `BOT_TOKEN` (без пробілів, без лапок у значенні).
2. Відкрий **Deploy Logs** для останнього деплою (ce4fe60d) і скопіюй усі рядки після "Starting container" або будь-який Traceback.
3. Якщо логів майже немає — контейнер падає дуже швидко; тоді найімовірніше саме **BOT_TOKEN** (або помилка імпорту/конфігу).
4. Змінив Variables → збережи → зроби **Redeploy** (без зміни коду), щоб контейнер перезапустився з новими змінними.

## 5. Перевірка локально з такими ж змінними

```bash
cd frogbot
export BOT_TOKEN="твій_токен"
python seed.py && python main.py
```

Якщо тут все працює, а на Railway ні — проблема в змінних оточення або в тому, з якої директорії Railway запускає контейнер (Root Directory).
